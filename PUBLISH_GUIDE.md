# Публикация «Дело» в Play Market и App Store

Полное руководство с учётом российских реалий.

> **Подробная пошаговая инструкция со ссылками и рекомендации по рекламе:** см. файл **ИНСТРУКЦИЯ_ПУБЛИКАЦИЯ_И_РЕКЛАМА.md**.

---

## Российские реалии — кратко

| Вопрос | Google Play | App Store |
|--------|-------------|-----------|
| **Регистрация** | $25 один раз | $99 в год |
| **Оплата из РФ** | Карты МИР/Сбер/ВТБ часто не проходят; пробуют VPN, зарубежные карты или родственников за границей | То же: зарубежная карта или платёж из-за рубежа |
| **Скачивание в РФ** | Работает: пользователи из России могут ставить бесплатные приложения | Работает |
| **In-app покупки в РФ** | Ограничены с 2022 года | Могут быть ограничены. «Дело» бесплатное — не касается |
| **Аккаунт разработчика** | Физлицо — нужен ИП или самозанятый (бизнес-аккаунт) | Apple принимает физлицо; нужен Mac + Xcode для сборки iOS |

**Итог:** бесплатное приложение можно публиковать и распространять в РФ; главная сложность — оплата взноса разработчика (решают зарубежной картой или через посредников).

## Часть 1. Google Play

### 1.1 Регистрация и доступ

| Шаг | Действие |
|-----|----------|
| 1 | Зарегистрировать **Google Play Console** — $25 единоразово (оплата картой Visa/Mastercard, работает из РФ) |
| 2 | Создать **бизнес-аккаунт разработчика** (для физлица — ИП или самозанятый) |
| 3 | Подтвердить личность (паспорт) — обычно занимает 1–2 дня |

**Российские реалии:**
- Оплата $25 проходит через карты МИР/ВТБ/Сбер — могут быть ограничения; часто используют VPN и зарубежные карты.
- С 2022 года Google ограничил приём платежей от пользователей в РФ (in-app purchases). Ваше приложение бесплатное — это не затрагивает.
- Раздача приложений в РФ работает: пользователи из России могут скачивать бесплатные приложения.

### 1.2 Что подготовить для Google Play

| Элемент | Требования | Где используется |
|---------|------------|------------------|
| **APK/AAB** | Подписанный bundle (AAB) | Загрузка в консоль |
| **Название** | до 30 символов | «Дело» |
| **Краткое описание** | до 80 символов | Карточка в магазине |
| **Полное описание** | до 4000 символов | Страница приложения |
| **Скриншоты** | минимум 2, тел. 320–3840px | Обязательно |
| **Иконка** | 512×512 px, PNG, 32-bit | Магазин |
| **Промо-графика** | 1024×500 px | По желанию |
| **Политика конфиденциальности** | URL (обязательно) | Консоль |
| **Категория** | Productivity / Производительность | Консоль |
| **Контент-рейтинг** | Анкета (обычно «Для всех») | Обязательно |
| **Целевая аудитория** | Возраст | Обязательно |

### 1.3 Сборка AAB для Google Play

```
cd DeloApp/frontend
npm run cap:sync
npx cap open android
```

В Android Studio:
1. **Build → Generate Signed Bundle / APK**
2. Выбрать **Android App Bundle (AAB)**
3. Создать keystore (сохранить пароль и alias)
4. Выбрать release
5. Собрать → получится `app-release.aab`

## Часть 2. App Store (Apple)

### 2.1 Регистрация и доступ

| Шаг | Действие |
|-----|----------|
| 1 | Зарегистрировать **Apple Developer Program** — $99/год |
| 2 | Нужен **Mac** с Xcode (для iOS-сборки) |
| 3 | Подтверждение личности и юридического лица |

**Российские реалии:**
- Оплата $99 — карты РФ могут не приниматься; часто используют зарубежные карты или родственников за границей.
- App Store в РФ работает, пользователи могут скачивать приложения.
- In-app purchases для российских пользователей могут быть ограничены.

### 2.2 Что подготовить для App Store

| Элемент | Требования |
|---------|------------|
| **IPA** | Собранный и подписанный в Xcode |
| **Название** | до 30 символов |
| **Подзаголовок** | до 30 символов |
| **Описание** | до 4000 символов |
| **Скриншоты** | по размерам устройств (iPhone 6.7", 6.5", 5.5" и т.д.) |
| **Иконка** | 1024×1024 px, PNG, без прозрачности |
| **Политика конфиденциальности** | URL (обязательно) |
| **Категория** | Productivity |
| **Возрастной рейтинг** | Заполнить анкету |

### 2.3 Сборка для App Store

```
cd DeloApp/frontend
npx cap add ios   # если ещё не добавлено
npm run cap:sync
npx cap open ios
```

В Xcode:
1. Выбрать Team (подписка разработчика)
2. Product → Archive
3. Distribute App → App Store Connect

## Часть 3. Что ещё нужно для релиза

### 3.1 Сайт (landing / Privacy Policy)

**Зачем:**
- Play Market и App Store **обязательно** требуют URL политики конфиденциальности.
- Сайт повышает доверие и даёт ссылку для маркетинга.

**Минимум:**
- Одна страница с **политикой конфиденциальности**.
- Можно добавить описание приложения, скриншоты, ссылки на магазины.

**Где разместить:**
- GitHub Pages (бесплатно)
- Vercel / Netlify (бесплатно)
- Свой хостинг

**Что могу предложить:**
- Текст политики конфиденциальности под ваше приложение (данные только в localStorage, без аккаунта, без сервера).
- HTML-страницу с политикой и базовой разметкой.
- Простой landing на одной странице.

---

### 3.2 Логотип приложения

**Требования:**

| Платформа | Размер | Формат |
|-----------|--------|--------|
| Google Play | 512×512 px | PNG, 32-bit |
| App Store | 1024×1024 px | PNG, без прозрачности |
| Android (разные плотности) | 48–192 px | PNG |

**Что могу предложить:**
- Описание логотипа для генерации (в Cursor/нейросети): «Минималистичная иконка для приложения задач „Дело“ — галочка в круге, зелёный акцент, тёмный фон».
- Экспорт в нужные размеры — можно сделать через ImageMagick/скрипты после того, как будет базовое изображение.

---

### 3.3 Favicon для сайта

**Размеры:**
- 16×16, 32×32, 48×48 (классические)
- 180×180 (Apple Touch Icon)
- 192×192, 512×512 (PWA)

**Что могу предложить:**
- HTML-разметка с несколькими размерами favicon.
- SVG-иконка, масштабируемая под любые размеры.

---

### 3.4 Скриншоты для магазинов

**Google Play:** минимум 2, рекомендовано 4–8.  
**App Store:** по каждому размеру устройства (обычно 3–6 вариантов).

**Идеи:**
- Экран «Сегодня» со списком задач
- Экран с напоминанием
- Темная тема
- Прогресс-бар выполнения

**Что могу предложить:**
- Список сцен для скриншотов с текстами.
- Простой шаблон/инструкцию по оформлению (рамки, подписи) в Figma или Canva.

---

### 3.5 Тексты для магазинов

**Краткое описание (до 80 символов):**
> Задачи на сегодня и завтра. Выгрузи дела из головы.

**Полное описание (шаблон):**
> «Дело» — простое приложение для задач на сегодня и завтра.  
> 
> • Добавляй задачи и отмечай выполненные  
> • Устанавливай напоминания (09:00, 13:00, 15:00 или своё время)  
> • Переноси невыполненное на завтра  
> • Тёмная и светлая тема  
> • Работает офлайн, данные хранятся только на устройстве  

**Что могу предложить:**
- Доработать эти тексты под ваш стиль.
- Написать вариант на английском для международной аудитории.

---

### 3.6 Политика конфиденциальности

**Обязательные пункты для «Дело»:**
- Какие данные собираются (ничего, кроме localStorage на устройстве)
- Нет аккаунта, нет сервера, нет аналитики
- Ссылка на поддержку (https://delodelai.ru)

**Что могу предложить:**
- Готовый текст политики конфиденциальности на русском.
- HTML-страницу с этой политикой для размещения на сайте.
