## Сборка AAB для Android (локально)

1. Установите **JDK 17 или новее** и выставьте `JAVA_HOME` (сейчас в PATH часто остаётся Java 8 — Gradle Expo 54 **не соберётся** на JVM 8).
2. В каталоге `mobile-expo`:
   ```bash
   npm install
   npx expo prebuild --platform android
   cd android
   .\gradlew.bat bundleRelease
   ```
   На macOS/Linux: `./gradlew bundleRelease` вместо `gradlew.bat`.

3. **Готовый файл AAB** (после успешной сборки):

   `mobile-expo/android/app/build/outputs/bundle/release/app-release.aab`

**Важно для Google Play:** в шаблоне Expo `release` может быть подписан **debug-keystore** — такой AAB подходит для внутреннего теста, но **для публикации в прод** настройте подпись релизным ключом (Android Studio → Generate Signed Bundle, или **EAS Build** с учётными данными Play App Signing).

### Альтернатива: облачная сборка

```bash
cd mobile-expo
npx eas-cli build --platform android --profile production
```

Скачанный `.aab` появится в артефактах сборки на expo.dev (удобно, если локально нет JDK 17).

## App Store (технически)

- Версия в приложении: `app.json` → `expo.version` (**1.1.0**), `ios.buildNumber` (**10**).
- Загрузка: **EAS Submit** или архив в **Xcode** после `npx expo prebuild --platform ios` (на Mac) с настроенной командой разработчика.

Папка `mobile-expo/android` в репозитории в `.gitignore` — её создаёт `expo prebuild`; путь к AAB выше актуален **на вашей машине** после сборки.
