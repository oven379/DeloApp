# Веб-версия приложения «Дело» (Windows, Linux, iOS в браузере)

Приложение уже собрано как **веб-приложение** (React + Vite). Один и тот же код идёт в браузер и в оболочки Capacitor (Android/iOS). Чтобы получить «веб-версию» для **Windows, Linux и iOS** (открытие в браузере), нужно в основном **разместить сборку в интернете** и при желании немного доработать.

---

## Что уже есть

| Элемент | Статус |
|--------|--------|
| Сборка для браузера | Есть: `npm run build` → папка `frontend/dist/` |
| Работа в браузере | Есть: логика не завязана только на Capacitor |
| Уведомления в браузере | Есть: в `notifications.js` используется стандартный Notification API, если не нативное приложение |
| PWA (манифест + sw) | Есть: `manifest.json`, `sw.js`, в `main.jsx` регистрация Service Worker |
| Адаптивная вёрстка | Есть: тот же интерфейс, что и в мобильном приложении |

То есть **веб-версия для Windows, Linux и iOS (Safari)** — это по сути та же сборка, выложенная на хостинг и открытая в браузере.

---

## Что нужно сделать

### 1. Разместить веб-приложение в интернете

Собранная папка `frontend/dist/` должна отдаваться по HTTPS.

**Варианты:**

- **Поддомен:** например `app.delodelai.ru` — указать в DNS корень домена на хостинг и отдавать содержимое `dist/` как статику.
- **Подпапка на текущем сайте:** например `https://delodelai.ru/app/`. Тогда в `frontend/vite.config.js` задать `base: '/app/'`, пересобрать (`npm run build`) и залить содержимое `dist/` в каталог `/app/` на сервере (или в репозиторий лендинга в папку `app/` и настроить отдачу).

После этого:

- **Windows / Linux:** открыть в Chrome, Edge, Firefox по ссылке (например `https://app.delodelai.ru` или `https://delodelai.ru/app/`).
- **iOS:** открыть ту же ссылку в **Safari** — это и есть «веб-версия для iOS».

Дополнительно в PWA:

- На **десктопе** (Chrome/Edge): можно нажать «Установить приложение» / «Install app» и использовать как отдельное окно.
- На **iOS**: в Safari «Добавить на экран „Домой“» — иконка как у приложения, открывается в полноэкранном режиме без адресной строки.

---

### 2. (Опционально) Настроить base и пути

Если приложение открывается по подпапке (например `https://delodelai.ru/app/`):

1. В **`frontend/vite.config.js`** указать:
   ```js
   base: '/app/',
   ```
2. Пересобрать: `npm run build`.
3. Убедиться, что на сервере все запросы внутри `/app/` отдаются `index.html` (SPA fallback), иначе при прямом заходе по `.../app/calendar` будет 404. На GitHub Pages для подпапки это делается через `404.html` с редиректом на `index.html` (можно описать отдельно).

Если приложение на отдельном домене/поддомене (например `app.delodelai.ru`) с корня, оставляете `base: './'` — ничего менять не нужно.

---

### 3. (Опционально) Отдельное «десктопное» приложение (Windows / Linux)

Если нужна именно **программа** с иконкой в меню и установкой (.exe / .AppImage), а не только сайт в браузере:

- **Electron** или **Tauri**: отдельный минимальный проект, который открывает в окне либо URL веб-версии, либо локальную папку `dist/`. Тогда пользователь качает установщик под Windows/Linux и получает окно приложения без браузера.
- Это уже не «веб-версия», а десктопная оболочка вокруг той же веб-сборки; объём работ — настройка Electron/Tauri, иконки, подпись, дистрибуция.

Для «веб-версии для Windows и Linux» достаточно **п. 1** (размещение и открытие в браузере).

---

### 4. Поведение на веб и отличия от нативного приложения

- **Уведомления:** в браузере работают через Notification API (уже реализовано в `notifications.js`). Пользователь один раз разрешает уведомления для сайта. В iOS Safari ограничения такие же, как у любого веб-сайта (нужно разрешение, могут не показываться при полном закрытии Safari — это ограничение ОС).
- **Ежедневные напоминания «Утро»/«Вечер»:** на нативных платформах ставятся через Capacitor Local Notifications; в веб-версии этот код не выполняется (`if (!isNative()) return`). При желании можно добавить для веба расписание через `setInterval` или однократные `setTimeout` при открытой вкладке — полный аналог «в фоне» в браузере невозможен.
- **Данные:** как и в мобильном приложении, всё в `localStorage` на устройстве; при очистке данных сайта в браузере список дел сбрасывается — это ожидаемо для веб-версии.

---

## Краткий чек-лист «веб-версия для Windows, Linux и iOS»

1. Выбрать способ размещения: поддомен (`app.delodelai.ru`) или подпапка (`delodelai.ru/app/`).
2. При подпапке — выставить `base: '/app/'` в Vite и настроить SPA fallback на сервере.
3. Собрать: `cd frontend && npm run build`.
4. Залить содержимое `frontend/dist/` на хостинг в выбранное место.
5. Проверить в браузере: Windows/Linux (Chrome, Edge, Firefox) и iOS (Safari).
6. При желании: проверить PWA — «Установить приложение» на десктопе и «Добавить на экран „Домой“» на iPhone.

После этого веб-версия для **Windows, Linux и iOS (в браузере)** будет готова. Отдельные десктопные установщики (Electron/Tauri) — по желанию, как следующий шаг.
