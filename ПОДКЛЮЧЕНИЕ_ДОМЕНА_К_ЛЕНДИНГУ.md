# Как подключить лендинг к доменному адресу (delodelai.ru)

Инструкция: где разместить папку `landing/` и как привязать к ней ваш домен. Все варианты дают **HTTPS** автоматически.

---

## Общая схема

1. **Зарегистрировать домен** (если ещё нет) — у регистратора: Reg.ru, Timeweb, Cloudflare, Namecheap и т.д.
2. **Разместить лендинг** на хостинге (Vercel, Netlify, GitHub Pages или обычный хостинг).
3. **Настроить DNS** — указать домену, куда идти (на серверы хостинга).

Ниже — по шагам для каждого варианта хостинга.

---

## Вариант A: Vercel (удобно, бесплатно)

Подходит для статики (HTML/CSS/JS). Автоматический HTTPS.

### Шаг 1. Регистрация и деплой

| Действие | Ссылка |
|----------|--------|
| Зарегистрироваться | https://vercel.com/signup |
| Документация по доменам | https://vercel.com/docs/concepts/projects/domains |

**Как загрузить лендинг:**

- **Через Git:** создайте репозиторий (GitHub/GitLab/Bitbucket), залейте туда папку `landing` (или весь проект с `landing` в корне). В Vercel: **Add New Project** → импорт из Git → укажите **Root Directory** = `landing`, **Build Command** оставьте пустым, **Output Directory** = `.` или оставьте пустым (статический экспорт).
- **Через Vercel CLI:** установите CLI, в папке `landing` выполните `vercel` и следуйте подсказкам.

После деплоя сайт будет доступен по адресу вида `ваш-проект.vercel.app`.

### Шаг 2. Добавить свой домен в Vercel

1. Откройте проект в Vercel: https://vercel.com/dashboard  
2. **Settings** → **Domains** → **Add** → введите `delodelai.ru` и (по желанию) `www.delodelai.ru`.  
3. Vercel покажет, **какие DNS-записи** нужно создать у регистратора.

### Шаг 3. Настроить DNS у регистратора домена

Зайдите в панель того, у кого куплен домен (Reg.ru, Timeweb, Cloudflare, Namecheap и т.п.) → раздел **DNS / Управление зоной / DNS-записи**.

**Если домен куплен не в Vercel**, добавьте записи так:

| Тип  | Имя (Host) | Значение (Value)      | Примечание        |
|------|------------|------------------------|-------------------|
| **A**   | `@`        | `76.76.21.21`          | для delodelai.ru  |
| **CNAME** | `www`    | `cname.vercel-dns.com` | для www.delodelai.ru |

Часть регистраторов для корня домена поддерживают **ALIAS/ANAME** вместо A — тогда можно указать `cname.vercel-dns.com`. Точные подсказки смотрите в Vercel в разделе **Domains** после добавления домена.

### Шаг 4. Дождаться обновления DNS и HTTPS

- Обновление DNS: обычно **15–60 минут**, иногда до **24–48 часов**.  
- HTTPS: Vercel сам выпустит сертификат после того, как DNS начнёт указывать на Vercel.  
- Проверка распространения DNS: https://www.whatsmydns.net  

**Официальная справка Vercel по доменам:**  
https://vercel.com/docs/concepts/projects/domains  

**Добавить домен (пошагово):**  
https://vercel.com/docs/concepts/projects/configure-a-custom-domain

## Вариант B: Netlify (удобно, бесплатно)

Тоже статика, автоматический HTTPS.

### Шаг 1. Регистрация и деплой

| Действие | Ссылка |
|----------|--------|
| Регистрация | https://app.netlify.com/signup |
| Управление доменами | https://docs.netlify.com/domains-https/custom-domains/ |

**Как загрузить лендинг:**

- **Drag & Drop:** https://app.netlify.com — перетащите папку `landing` в зону «Deploy manually».  
- **Через Git:** как в Vercel — репозиторий с папкой `landing`, в настройках сборки указать **Publish directory** = `landing` (или корень, если в репо только содержимое лендинга).

Сайт получит адрес вида `имя-проекта.netlify.app`.

### Шаг 2. Добавить домен в Netlify

1. **Site configuration** → **Domain management** → **Add custom domain** / **Add a domain**.  
2. Введите `delodelai.ru` и при желании `www.delodelai.ru`.  
3. Netlify подскажет, какие записи создать.

### Шаг 3. DNS у регистратора

**Вариант 1 — оставляете DNS у регистратора.** Создайте записи:

| Тип    | Имя (Host) | Значение                         |
|--------|------------|-----------------------------------|
| **A**    | `@`        | `75.2.60.5`                       |
| **CNAME** | `www`    | `ваш-сайт.netlify.app` (имя из Netlify) |

**Вариант 2 — перенос DNS в Netlify (Netlify DNS).**  
В Netlify: **Domain management** → для домена выберите **Set up Netlify DNS**. Netlify даст вам **4 NS-сервера**. В панели регистратора домена замените текущие NS на эти четыре — тогда все записи можно править уже в Netlify, HTTPS выставится автоматически.

**Документация Netlify:**  
- Настройка внешнего DNS: https://docs.netlify.com/domains-https/custom-domains/configure-external-dns/  
- Настройка Netlify DNS: https://docs.netlify.com/domains/set-up-netlify-dns/

## Вариант C: GitHub Pages (бесплатно)

Удобно, если проект уже в GitHub.

### Шаг 1. Репозиторий и включение Pages

1. Создайте репозиторий на https://github.com  
2. Залейте в него содержимое папки `landing` (можно в корень или в папку `landing`).  
3. **Settings** → **Pages** → **Source**: выберите ветку (например, `main`) и папку с лендингом (например, `/` или `/landing`).  
4. Сохраните. Сайт будет по адресу `ваш-логин.github.io/имя-репо` или `ваш-логин.github.io`, если репозиторий специальный (`username.github.io`).

### Шаг 2. Свой домен

1. В корне репозитория создайте файл **CNAME** (без расширения), внутри одна строка:  
   `delodelai.ru`  
2. В **Settings** → **Pages** в поле **Custom domain** введите `delodelai.ru` и сохраните.

### Шаг 3. DNS у регистратора

| Тип  | Имя | Значение |
|------|-----|----------|
| **A**    | `@`  | `185.199.108.153` |
| **A**    | `@`  | `185.199.109.153` |
| **A**    | `@`  | `185.199.110.153` |
| **A**    | `@`  | `185.199.111.153` |
| **CNAME** | `www` | `ваш-логин.github.io` (или ваш-логин.github.io/имя-репо, если сайт в подпапке) |

**Справка GitHub Pages по кастомному домену:**  
https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site

## Вариант D: Обычный хостинг (Reg.ru, Timeweb, Beget и т.д.)

Если домен и хостинг уже есть у одного провайдера:

1. Откройте **файловый менеджер** или подключитесь по **FTP/SFTP**.  
2. Зайдите в каталог сайта (часто `public_html` или `www` для домена).  
3. Загрузите **все файлы** из папки `landing` (index.html, папки my-story и privacy с index.html внутри, styles.css, папка screenshots, favicon, logo.png и т.д.) в этот каталог.  
4. Домен уже привязан к хостингу в панели. **SSL:** включите «Let’s Encrypt» или «SSL-сертификат» в настройках домена на хостинге.

Ссылки на справку по загрузке сайта обычно в разделе «Хостинг» или «Файлы» на сайте вашего провайдера.

---

## Проверка после настройки

1. Откройте в браузере: `https://delodelai.ru` и `https://delodelai.ru/privacy.html`.  
2. Убедитесь, что в адресной строке есть замочек (HTTPS).  
3. Для Play Market и App Store нужен именно **HTTPS**-адрес политики конфиденциальности, например:  
   `https://delodelai.ru/privacy.html`
