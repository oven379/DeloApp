# Подробная инструкция: публикация приложения «Дело» и реклама

Краткую техническую часть смотри в **PUBLISH_GUIDE.md** и **STORE_READINESS.md**. Здесь — пошагово с ссылками и варианты рекламы.

---

## Часть 1. Публикация в Google Play (Android)

### Шаг 1. Аккаунт разработчика

| Действие | Ссылка | Примечание |
|----------|--------|------------|
| Зарегистрироваться в Google Play Console | https://play.google.com/console/signup | Нужен обычный Google-аккаунт |
| Оплатить регистрационный взнос | $25 один раз | В консоли после регистрации будет предложено оплатить. Карты РФ (МИР, Сбер, ВТБ) иногда не проходят — пробуют VPN или зарубежную карту |

**Для физлица (самозанятый/ИП):** при создании аккаунта выберите тип «Индивидуальный разработчик» или «Организация». Для России часто указывают ИП или самозанятого; подтверждение личности по паспорту — 1–2 дня.

### Шаг 2. Подготовка приложения

| Что | Где в проекте | Комментарий |
|-----|----------------|-------------|
| Сборка AAB | `frontend/` → Android Studio → Build → Generate Signed Bundle/APK | Подробно в **PUBLISH_GUIDE.md**, раздел 1.3 |
| Версия | `frontend/android/app/build.gradle` — `versionCode`, `versionName` | Перед каждой загрузкой увеличивать versionCode |
| Keystore | Создаётся при первой подписанной сборке | **Обязательно сохранить** файл .jks и пароли — без них нельзя обновлять приложение |

### Шаг 3. Создание приложения в консоли

1. Войдите: https://play.google.com/console  
2. **«Создать приложение»** → название «Дело», язык по умолчанию (русский).  
3. Заполните **«Политика конфиденциальности»** — URL вашей страницы, например:  
   `https://delodelai.ru/privacy.html`  
4. Укажите **категорию**: «Продуктивность» (Productivity).

### Шаг 4. Контент приложения (Store listing)

| Поле | Требование | Пример для «Дело» |
|------|------------|-------------------|
| Краткое описание | до 80 символов | Задачи на сегодня и завтра. Выгрузи дела из головы. |
| Полное описание | до 4000 символов | Текст есть в **PUBLISH_GUIDE.md**, раздел 3.5 |
| Графика: иконка | 512×512 px, PNG 32-bit | Из проекта: логотип в нужном размере |
| Графика: скриншоты | минимум 2, макс. 8; телефон 320–3840 px | Уже есть в `landing/screenshots/` |
| Промо-графика | 1024×500 px | По желанию |

Официальные требования к графике:  
https://support.google.com/googleplay/android-developer/answer/9866151

### Шаг 5. Контент-рейтинг и аудитория

1. В меню приложения: **«Политика»** → **«Контент приложения»** → **«Рейтинг контента»**.  
2. Заполните анкету (для «Дело» обычно: нет насилия, рекламы, покупок → **«Для всех»**).  
3. **«Целевая аудитория»** — укажите возраст (например, 18+ или без ограничений, по вашей политике).

### Шаг 6. Загрузка AAB и отправка на проверку

1. **«Выпуск»** → **«Производство»** (или тестовый трек) → **«Создать новый выпуск»**.  
2. Загрузите файл **app-release.aab**.  
3. Введите описание изменений (например: «Первая версия»).  
4. **«Отправить на проверку»**.

Проверка обычно 1–7 дней. После одобрения приложение появится в Google Play.

**Полезные ссылки:**

- Справка Play Console: https://support.google.com/googleplay/android-developer  
- Требования к приложению: https://support.google.com/googleplay/android-developer/answer/10144311

## Часть 2. Публикация в App Store (iOS)

### Шаг 1. Аккаунт разработчика Apple

| Действие | Ссылка | Примечание |
|----------|--------|------------|
| Зарегистрироваться в Apple Developer Program | https://developer.apple.com/programs/enroll/ | $99 в год. Оплата картой; из РФ часто используют зарубежную карту |
| Нужен Mac с Xcode | https://developer.apple.com/xcode/ | Сборка iOS возможна только на macOS |

### Шаг 2. Подготовка проекта iOS

В проекте уже есть `frontend/ios/`. На Mac:

```bash
cd frontend
npm run build
npx cap sync ios
npx cap open ios
```

В Xcode: выберите **Team** (ваш Apple Developer аккаунт), при необходимости включите **Signing & Capabilities** (например, Push Notifications для локальных уведомлений).

### Шаг 3. App Store Connect

| Действие | Ссылка |
|----------|--------|
| Войти в App Store Connect | https://appstoreconnect.apple.com |
| Создать приложение | «Мои приложения» → «+» → «Новое приложение» |
| Указать политику конфиденциальности | URL: https://delodelai.ru/privacy.html |

Заполните: название «Дело», подзаголовок, описание, категорию «Productivity», скриншоты по размерам iPhone (6.7", 6.5", 5.5" и т.д.). Требования:  
https://developer.apple.com/app-store/product-page/

### Шаг 4. Сборка и загрузка IPA

1. В Xcode: **Product → Archive**.  
2. После архивации: **Distribute App** → **App Store Connect** → загрузить.  
3. В App Store Connect в разделе **«Тестирование»** выберите загруженную сборку и отправьте на **«Отправить на проверку»**.

Проверка Apple обычно 1–3 дня, иногда дольше.

**Полезные ссылки:**

- Руководство по публикации: https://developer.apple.com/app-store/submitting/  
- App Store Connect: https://developer.apple.com/support/app-store-connect/

## Часть 3. Перед публикацией — чек-лист

- [ ] Сайт delodelai.ru доступен по HTTPS, на нём открывается страница политики: `https://delodelai.ru/privacy.html`  
- [ ] В консолях Play и App Store указан именно этот URL политики  
- [ ] Версия и versionCode/versionName обновлены в проекте  
- [ ] Кнопки «Скачать в Google Play» и «Скачать в App Store» на лендинге ведут на реальные ссылки магазинов (после публикации подставьте свои URL)

После публикации замените на лендинге `href="#"` у кнопок магазинов на:

- Google Play: `https://play.google.com/store/apps/details?id=app.delodelai` (или ваш applicationId)  
- App Store: `https://apps.apple.com/app/idXXXXXXXXX` (ID приложения из App Store Connect)

## Часть 4. Реклама: что предложить

### 4.1 Приоритет 1: бесплатно и быстро

| Канал | Что сделать |
|-------|-------------|
| **Яндекс.Метрика на сайте** | Подключить на delodelai.ru (в политике вы уже указали возможность Метрики). Смотреть переходы, отказы, откуда приходят пользователи. |
| **Telegram** | Написать о приложении в своих каналах/чатах, попросить друзей и 379team распространить. Создать канал или чат «Дело» для обратной связи и анонсов. |
| **ASO (оптимизация в магазинах)** | В описаниях и подзаголовках использовать ключевые слова: «задачи», «напоминания», «список дел», «без рекламы», «офлайн». Это помогает находить приложение по поиску в Play и App Store. |

### 4.2 Приоритет 2: платная реклама с небольшим бюджетом

| Платформа | Ссылка | Для кого | Ориентир |
|-----------|--------|----------|----------|
| **Яндекс.Директ (поиск + РСЯ)** | https://direct.yandex.ru | Поиск по запросам «приложение для дел», «напоминания задачи» и т.п.; реклама на сайтах партнёров | Старт от 300–500 ₽/день, таргетинг по РФ |
| **VK Реклама** | https://vk.com/ads | Таргетинг по интересам (продуктивность, саморазвитие, планирование), возраст, гео | Старт от 200–500 ₽/день |
| **Telegram Ads** | https://promote.telegram.org | Показ в каналах Telegram (если канал подключён к рекламной платформе) | Зависит от канала, обычно от 50–100 € за размещение |

### 4.3 Приоритет 3: когда будет стабильный бюджет

| Канал | Зачем |
|-------|--------|
| **Google Ads (Управление рекламой приложений)** | https://ads.google.com — кампании по установкам приложения (UAC). Показ в поиске, YouTube, в других приложениях. Эффективно, но бюджет обычно от 500–1000 ₽/день. |
| **Яндекс.Директ — кампания на приложение** | В Директе можно настроить кампанию на установки мобильного приложения (если приложение привязано к счётчику Метрики). |

### 4.4 Что не забыть

- На лендинге уже есть история «История одного дела», поддержка в Telegram, призыв писать идеи и донаты — это хорошо для сарафанного радио.  
- После подключения Яндекс.Метрики добавьте в политику конфиденциальности точную формулировку (что собирается, зачем, ссылка на политику Яндекса).  
- Рекламу лучше тестировать небольшими суммами: запустить на 3–5 дней, посмотреть переходы на сайт и установки (через Метрику и отчёты в рекламных кабинетах), потом масштабировать.

## Часть 5. Проверка кода лендинга

Проведена проверка файлов `landing/` (index.html, my-story/index.html, privacy/index.html, styles.css):

- **Критических ошибок и мёртвого кода не найдено.** Линтер ошибок не показывает.  
- Все используемые классы (scroll-progress, nav-back, nav-link--story, nav-link--support и др.) задействованы в разметке или в стилях.  
- Кнопки «Google Play» и «App Store» сейчас ведут на `href="#"` — это временно; после публикации подставьте реальные ссылки на страницы приложения в магазинах (см. Часть 3).

При необходимости можно дополнительно минифицировать CSS или отключить неиспользуемые медиа-запросы — для текущего объёма лендинга это не обязательно.

---

**Итог:** сначала опубликуйте приложение по шагам 1–3, затем подключите Метрику и по желанию запустите рекламу по приоритетам 1–2. Ссылки на консоли и справку выше — актуальные на момент составления инструкции.
