# Запуск и сборка DeloApp

DeloApp — это **React + Vite + Capacitor**. Для сборки под Android **Flutter не нужен**. Нужны: Node.js, Android Studio (или JDK + SDK).

---

## Нужные папки

| Папка | Назначение |
|-------|------------|
| **DeloApp** | Корень проекта |
| **DeloApp/frontend** | Веб-приложение (React), здесь запускают `npm` |
| **DeloApp/frontend/android** | Проект Android (Capacitor), открывать в Android Studio |
| **DeloApp/frontend/dist** | Собранный веб-билд (появляется после `npm run build`) |
| **DeloApp/app** | Бэкенд Laravel (пока не обязателен для сборки) |

Открыть все эти папки в проводнике: **дважды запусти `открыть_папки.bat`** в корне DeloApp.

---

## Быстрый запуск (разработка в браузере)

```bash
cd frontend
npm install
npm run dev
```

Открой в браузере: **http://localhost:5173**

---

## Сборка и запуск на Android

### Вариант 1: скрипты (проще всего)

В папке **DeloApp** запусти:

- **run.bat** (двойной клик) или в PowerShell: **.\run.ps1**

Скрипт сделает: `npm install` → `npm run build` → `npx cap sync`. После этого открой Android Studio:

- Запусти **открыть_android_studio.bat** в папке DeloApp  
  **или** в папке `frontend` выполни: **npx cap open android**

В Android Studio выбери устройство/эмулятор и нажми **Run** (▶).

### Вариант 2: вручную

```bash
cd c:\Users\PC\Desktop\projects\cursor\DeloApp\frontend
npm install
npm run build
npx cap sync
npx cap open android
```

В открывшейся Android Studio: Run на устройстве или эмуляторе.

## Как увидеть изменения

- **В браузере:** запущен `npm run dev` — после правок в `frontend/src` страница обновится сама (hot reload). Код берётся из исходников.
- **На эмуляторе/телефоне:** приложение показывает **собранный** код из папки `dist/`. Чтобы увидеть последние правки:
  1. Выполни в корне DeloApp **run.bat** (или в `frontend`: `npm run build` и `npx cap sync`).
  2. В Android Studio нажми **Run** (▶).
  3. Если старый интерфейс всё ещё виден: меню **Build → Clean Project**, затем снова **Run**.

## Если понадобится Flutter

DeloApp Flutter **не использует**. Если Flutter нужен для другого проекта:

1. Скачай SDK: https://docs.flutter.dev/get-started (Custom setup → Install manually).
2. Распакуй в `C:\flutter`.
3. Добавь в PATH: **C:\flutter\bin** (Win + R → sysdm.cpl → Переменные среды → Path).

Подробнее: https://docs.flutter.dev/install/manual

## Готовность к релизу

1. **Версия:** в `frontend/package.json` поле `version` (например 1.0.0).
2. **Сборка:** из папки DeloApp выполни **run.bat** или **.\run.ps1** (build + cap sync).
3. **Android Studio:** открой **frontend/android**, Build → Generate Signed Bundle / APK, подпиши и собери AAB/APK.
4. **Чек-лист:** см. **RELEASE_CHECKLIST.md** — функции, настройки, иконки, политика конфиденциальности.
