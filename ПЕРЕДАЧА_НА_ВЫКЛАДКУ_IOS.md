# Передача кода для выкладки в App Store (Windows → Mac / Xcode)

Если вы разрабатываете на Windows, а сборку и загрузку в App Store делаете на Mac (или передаёте другому разработчику), используйте эту инструкцию.

**Рекомендуемый способ:** GitHub — один раз настроили репозиторий, дальше только `git push` и на Mac — `git pull` и сборка в Xcode.

---

## Вариант A: Через GitHub (рекомендуется)

### На Windows (вы): отправить код в GitHub

1. **Инициализировать Git** (если ещё не сделано):
   ```powershell
   cd c:\Users\PC\Desktop\projects\cursor\DeloApp
   git init
   git add .
   git commit -m "Дело: готово к сборке iOS"
   ```

2. **Создать репозиторий на GitHub:** https://github.com/new  
   - Название, например: `DeloApp`  
   - Без README (у вас уже есть файлы).  
   - Скопировать URL репозитория, например: `https://github.com/ВАШ_ЛОГИН/DeloApp.git`

3. **Привязать remote и отправить код:**
   ```powershell
   git remote add origin https://github.com/ВАШ_ЛОГИН/DeloApp.git
   git branch -M main
   git push -u origin main
   ```

4. **В дальнейшем**, после любых правок:
   ```powershell
   git add .
   git commit -m "Обновление для iOS"
   git push origin main
   ```

### На Mac: получить код и открыть в Xcode

1. **Клонировать репозиторий** (один раз):
   ```bash
   git clone https://github.com/ВАШ_ЛОГИН/DeloApp.git
   cd DeloApp/frontend
   ```

2. **Установить зависимости и собрать веб-часть:**
   ```bash
   npm install
   npm run build
   npx cap sync ios
   ```

3. **Открыть проект в Xcode:**
   ```bash
   npx cap open ios
   ```
   Или: в Finder открыть папку `DeloApp/frontend/ios/App` и дважды кликнуть по `App.xcworkspace` (именно **.xcworkspace**, не .xcodeproj).

4. **В Xcode:**
   - Выбрать **Team** (ваш Apple Developer аккаунт).
   - **Bundle Identifier** оставить `app.delodelai`.
   - Схема: **App** → устройство **Any iOS Device (arm64)**.
   - **Product** → **Clean Build Folder**, затем **Product** → **Archive**.

5. **После успешного Archive:**
   - Откроется **Organizer**.
   - Выбрать архив → **Distribute App** → **App Store Connect** → **Upload**.
   - Пройти мастер загрузки.

6. **При следующих обновлениях** (когда вы снова пушите в GitHub):
   ```bash
   cd /путь/к/DeloApp
   git pull origin main
   cd frontend
   npm run build
   npx cap sync ios
   npx cap open ios
   ```
   Дальше в Xcode снова **Product** → **Archive** и загрузка.

## Вариант B: ZIP-архив (если без GitHub)

### Step 1. Что подготовить (на Windows)

### 1.1. Pack the project

Create a ZIP archive of the **entire `frontend` folder**:

- Path: `c:\Users\PC\Desktop\projects\cursor\DeloApp\frontend`
- Include all contents: `src/`, `public/`, `ios/`, `android/`, `package.json`, `package-lock.json`, `capacitor.config.json`, etc.
- Exclude `node_modules` (it’s large; the developer will run `npm install`)
- Exclude `dist` (it will be generated by `npm run build`)

**Option A — Manual ZIP (without node_modules):**

1. Open `frontend` in File Explorer.
2. Select all except `node_modules` and `dist`.
3. Right-click → **Send to** → **Compressed (zipped) folder**.
4. Name it: `DeloApp-frontend-ios.zip`

**Option B — PowerShell:**

```powershell
cd c:\Users\PC\Desktop\projects\cursor\DeloApp
Compress-Archive -Path frontend\* -DestinationPath DeloApp-frontend-ios.zip -Force
```

This includes `node_modules`. If the archive is large, prefer Option A and send without `node_modules`.

## Step 3. Instructions for the Mac Developer

Copy this section and send it to your developer.

---

### 3.1. Requirements on Mac

- macOS
- [Xcode](https://developer.apple.com/xcode/) (from App Store)
- [Node.js](https://nodejs.org/) (LTS)
- Apple Developer account ($99/year) with access to the app

---

### 3.2. Setup and build

1. Extract the project (or clone the repo).
2. Open Terminal and run:

```bash
cd path/to/frontend
npm install
npm run build
npx cap sync ios
npx cap open ios
```

3. In Xcode:
   - Select **Team** (Apple Developer account).
   - Ensure **Bundle Identifier** is `app.delodelai`.
   - Select **Any iOS Device (arm64)** or a connected device.
   - **Product** → **Archive**.

4. In Organizer:
   - Select the new archive.
   - **Distribute App** → **App Store Connect** → **Upload**.
   - Follow the wizard.

5. In App Store Connect:
   - Open the app → version 1.0.
   - Under **Build**, click **+** and select the uploaded build.
   - Fill in metadata (see `ТЕКСТЫ_ДЛЯ_ВЫКЛАДКИ.md`).
   - **Add for Review** and submit.

## Замечание по экрану запуска (Launch Screen)

В проекте для iPad Multitasking используется **UILaunchScreen** в `frontend/ios/App/App/Info.plist` (ключ `UILaunchScreen` с изображением `Splash`). Это устраняет ошибку валидации App Store про launch storyboard. Менять не требуется.
